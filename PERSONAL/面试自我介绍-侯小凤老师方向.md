## 02｜我的研究兴趣表达

我希望未来围绕 **端侧/边缘场景下的高效智能计算系统** 开展研究，尤其关注：

1. **资源受限设备上的低时延 AI 推理**

   如何根据硬件资源约束调整模型结构、数据精度、执行路径和调度策略。

   例如 early-exit、动态精度、条件计算、传感器/模态门控等，让系统在不同场景下自动平衡精度与开销。

4. **从智能硬件到 AI 系统的工程闭环**  
   不只停留在算法指标，而是把采集、推理、调度、部署、测试与可视化串成可运行系统。

## 03｜核心项目一：基于 FPGA 的磁悬浮气隙辨识与无气隙传感器控制

**项目类型**：国家级 SRTP  
**时间**：2025.05—2026.05（预计）  
**关键词**：Zynq SoC｜Vitis HLS｜TCN｜实时推理｜FPGA 部署｜软硬件协同

### 项目背景

磁悬浮系统对气隙测量和实时控制要求很高。项目尝试利用电磁信号之间的冗余关系，构建无气隙传感器的气隙软测量方法，并进一步部署到 FPGA，实现面向实时控制的在线推理。

### 我的工作

- 设计并调试集成 **Zynq SoC、磁悬浮功率驱动、信号处理前端、ADC 采样电路** 的一体式实验主板。
- 在多种悬浮工况下采集实验数据，使用 **PyTorch** 训练 TCN 网络，实现气隙估计。
- 使用 **Vitis HLS** 将 TCN 模型部署到 Zynq SoC 的 PL 端，实现低时延在线推理。

### 关键结果

| 指标 | 结果 |
|---|---:|
| 气隙估计误差 | MAE = **1.364 ADC counts**，约 **0.011 mm** |
| 单次在线推理时延 | **17.24 μs** |
| 控制频率要求 | 满足 **10 kHz** 实时控制 |
| 学术产出 | 第一作者会议论文 IEEE ECIS 在投，扩刊论文计划投递 Measurement |

### 与侯老师方向的关联

这个项目可以自然引出我对 **边缘 AI 推理加速** 的兴趣：

```mermaid
flowchart LR
    A[实时控制需求] --> B[传感信号采集]
    B --> C[TCN模型训练]
    C --> D[HLS部署到Zynq PL]
    D --> E[17.24 μs在线推理]
    E --> F[精度/延迟/硬件资源权衡]
```

> [!tip] 面试表达重点
> 这个项目最能体现我对 **模型不是只在 GPU 上训练，而是要在真实受限硬件上稳定运行** 的理解。它与课题组关注的端侧推理、软硬件协同、实时 AIoT 系统非常接近。

项目面向锂电池安全与寿命评估，构建集阻抗测量、嵌入式采集、远程通信、可视化交互与 SOH 智能预测于一体的系统。

| 项目结题 | 国家级 SRTP **优秀结题** |

## 10｜面试前 3 天准备清单

### 必补概念

- Computer Architecture 基础：memory hierarchy、pipeline、parallelism、accelerator。
- AI inference 基础：latency、throughput、batching、quantization、operator、memory bandwidth。
- Edge AI 基础：功耗、实时性、资源约束、模型压缩、early-exit。
- 系统实验基础：profiling、benchmark、ablation study、performance counter。

### 建议重点读的论文题目

优先读摘要、引言、系统设计图和实验部分：

- **Architecting Efficient Multi-modal AIoT Systems**
- **MMExit: Enabling Fast and Efficient Multi-modal DNN Inference with Adaptive Network Exits**
- **SMG: A System-level Modality Gating Facility for Fast and Energy-Efficient Multimodal Computing**
- **CPM: A Cross-layer Power Management Facility to Enable Highly-efficient Real-time AIoT Systems**
- **SpaceExit: Enabling Efficient Adaptive Computing in Space with Early Exits**

### 个人项目准备

- 准备一张图解释磁悬浮项目链路：采样 → TCN → HLS → Zynq PL → 实时控制。
- 准备讲清楚为什么 TCN 适合时序信号，以及 FPGA 部署时最关注哪些资源。
- 准备承认短板：体系结构、分布式推理服务经验不足；但强调可快速学习并能做工程实验。
